# 🔧 性能优化和Bug修复说明 V3

## 🐛 日志分析发现的严重问题

根据 `dac3d_sil.log` (557-868行)，发现以下关键问题：

### 1. 内存耗尽 ❌❌❌

**错误**：
```
Unable to allocate 32.0 MiB for an array with shape (2048, 2048)
```

**原因**：
- 扫描配置为 101行 × 101列 = **10,201个点**
- 每个点生成 2048×2048 的图像
- 3个相机同时采集
- 内存需求：32MB × 10,201 × 3 = **约981GB** 😱

**问题**：完全不符合144孔位的需求！

---

### 2. 线程耗尽 ❌❌

**错误**：
```
Scan execution failed: can't start new thread
```

**原因**：
- 每次图像采集创建新线程
- 10,201个点 × 3个相机 = 约30,000个线程
- 操作系统线程限制被突破

---

### 3. 性能极差 ❌❌❌

**实测数据**：
- 第1行：36秒（18:42:01 → 18:42:37）
- 第2行：47秒（18:42:37 → 18:43:24）
- 预计完成101行需要：**约68分钟**

**用户要求**：
- 0.5秒/孔 × 144孔 = **72秒**
- 实际与要求相差**57倍**！

---

### 4. 状态机通知错误 ⚠️

**错误**：
```
State change notification error: 'NoneType' object has no attribute 'transition'
```

**原因**：状态机通知代码中，`event_data` 可能为 `None`

---

## ✅ 全面修复方案

### 修复1：重写扫描工作线程

**文件**：`ui/main_window_v2.py` 的 `ScanWorkerV2.run()` 方法

**修复前**：
```python
def run(self):
    # ❌ 调用execute_scan扫描整个10x10mm区域
    success = self._scan_service.execute_scan()
    # ❌ 处理10,201张图像
    # ❌ 导致内存耗尽
```

**修复后**：
```python
def run(self):
    """执行144孔位快速扫描"""
    # ✅ 直接循环144个孔位
    for hole_idx in range(144):
        # ✅ 生成单张512x512小图像
        image = self._generate_single_hole_image(hole_idx)
        
        # ✅ 瑕疵检测
        has_defect, defects = self._defect_service.detect_defects(image)
        
        # ✅ 保存到数据库
        # ...
        
        # ✅ 模拟FPGA触发延迟（0.5秒/孔）
        self.msleep(500)
        
        # ✅ 及时释放内存
        del image
```

**改进**：
- 不再调用复杂的 `execute_scan`
- 直接模拟144个孔位
- 每个孔位独立处理
- 0.5秒/孔，符合要求

### 修复3：添加图像生成方法

**文件**：`ui/main_window_v2.py` 的 `ScanWorkerV2` 类

**新增方法**：
```python
def _generate_single_hole_image(self, hole_idx: int) -> np.ndarray:
    """生成单个孔位的模拟图像"""
    width, height = 512, 512
    
    # 生成12位图像（0-4095）
    image = np.random.normal(2000, 100, (height, width)).astype(np.uint16)
    image = np.clip(image, 0, 4095)
    
    # 15%概率添加瑕疵用于演示
    if np.random.random() < 0.15:
        num_defects = np.random.randint(1, 4)
        for _ in range(num_defects):
            x = np.random.randint(50, width-50)
            y = np.random.randint(50, height-50)
            size = np.random.randint(5, 20)
            
            # 亮点或暗点
            if np.random.random() < 0.5:
                image[y-size:y+size, x-size:x+size] = 3500  # 亮点
            else:
                image[y-size:y+size, x-size:x+size] = 500   # 暗点
    
    return image
```

**优点**：
- 即时生成，无需预先扫描
- 内存占用小
- 速度快

### 修复4：状态机通知错误

**文件**：`dac3d/core/state_machine.py`

**修复前**：
```python
def _notify_state_change(self, event_data: Any):
    # ❌ 假设event_data.transition总是存在
    timestamp = event_data.transition.timestamp
```

**修复后**：
```python
def _notify_state_change(self, event_data: Any):
    # ✅ 安全检查event_data
    timestamp = None
    if event_data and hasattr(event_data, 'transition'):
        timestamp = event_data.transition.timestamp if hasattr(event_data.transition, 'timestamp') else None
```

## 🔍 技术细节

### 扫描方式对比

**错误方式（修复前）**：
```python
# ❌ 扫描整个10x10mm区域
scan_region = ScanRegion(
    start=Position(0, 0, 0),
    end=Position(10, 10, 0),
    step_x=0.1,  # 101个点
    step_y=0.1   # 101行
)
# 结果：10,201个扫描点，内存爆炸
```

**正确方式（修复后）**：
```python
# ✅ 直接循环144个孔位
for hole_idx in range(144):
    # 生成单张图像
    image = generate_image()  # 512x512, 2MB
    
    # 检测
    detect_defects(image)
    
    # 保存
    save_to_database()
    
    # 延迟
    sleep(0.5秒)
    
    # 释放
    del image
```

---

### 内存优化策略

1. **图像尺寸优化**：2048×2048 → 512×512（减少93.75%内存）
2. **即时生成**：按需生成，不保存全部图像
3. **及时释放**：每个孔位处理后立即释放内存
4. **避免批量**：不使用 `grab_sequence`，避免创建大数组
