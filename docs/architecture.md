# DAC-3D 系统架构文档

## 系统概述

DAC-3D是一个基于FPGA+运动控制卡的色散差动共聚焦产业化光学检测系统，采用五层架构设计，确保系统的稳定性、可扩展性和可维护性。

## 架构层次

### Layer 0: FPGA实时控制层

**职责**: 硬件实时触发控制，纳秒级精度

**核心模块**:
- `timing_ctrl.v`: 时序控制器顶层
- `pso_generator.v`: 位置同步输出生成器
- `encoder_decoder.v`: 编码器解码器(4倍频)
- `pwm_controller.v`: PWM触发信号生成
- `trigger_delay.v`: 触发延迟补偿

**技术特点**:
- Zynq-7020 SoC (FPGA + ARM)
- AXI-Lite总线接口
- 100MHz系统时钟
- <1ns时序精度

### Layer 1: 硬件抽象层(HAL)

**职责**: 统一硬件接口，实现设备热插拔

**核心接口** (`dac3d/hal/interfaces.py`):
- `IDevice`: 所有设备基类
- `IStage`: 运动台接口
- `ICamera`: 相机接口
- `IDMD`: DMD接口
- `ILight`: 光源接口
- `IFPGA`: FPGA控制器接口

**驱动实现**:
- `ZynqController`: FPGA控制器驱动
- `ZMotionStage`: ZMotion XY运动台驱动
- `PIPiezoStage`: PI压电Z轴驱动
- `BaslerCamera`: Basler相机驱动
- `CameraArray`: 相机阵列管理

**设计原则**:
1. 接口与实现分离
2. 支持模拟驱动(仿真测试)
3. 统一错误处理
4. 状态管理

### Layer 2: 核心层

**职责**: 系统核心逻辑，状态管理和事件分发

**核心模块**:

1. **状态机** (`dac3d/core/state_machine.py`)
   - 严格的状态转换控制
   - 13个状态: IDLE → INITIALIZING → HOMING → READY → ...
   - 基于transitions库实现
   - 防止非法状态转换

2. **事件总线** (`dac3d/core/event_bus.py`)
   - 发布-订阅模式
   - 模块间解耦通信
   - 支持同步/异步事件
   - 单例模式实现

3. **异常系统** (`dac3d/core/exceptions.py`)
   - 分层异常体系
   - 统一错误处理

### Layer 3: 服务层

**职责**: 业务逻辑封装，提供高级功能

**核心服务**:

1. **扫描服务** (`dac3d/services/scan_service.py`)
   - 协调硬件执行扫描
   - 硬触发飞拍扫描
   - 进度监控
   - 数据收集

2. **配置服务** (`dac3d/services/config_service.py`)
   - YAML配置管理
   - Pydantic参数验证
   - 配置热更新

3. **算法服务** (预留)
   - ARC标定
   - CFAN推理
   - 差动信号处理
   - 点云重建

4. **数据服务** (预留)
   - HDF5存储
   - 数据缓存
   - 日志管理

### Layer 4: 应用层

**职责**: 用户交互界面

**主界面** (`ui/main_window.py`):
- 基于PyQt5
- 控制面板: 系统控制、扫描控制
- 显示区域: 图像显示、3D显示、日志
- 状态栏和进度条
- 多线程扫描

**扩展**:
- Napari 3D插件(预留)
- 报表生成
- 远程监控

## 关键设计决策

### 1. 硬件同步触发

**问题**: 软件触发存在10-50ms随机抖动

**解决方案**:
```
ZMotion编码器位置 → FPGA PSO硬件比较 → TTL脉冲 → 同步触发
  ↓                                              ↓
编码器反馈                                    相机/DMD/LED
```

**优势**:
- 触发精度<10μs
- 位置与图像严格对齐
- 不依赖上位机实时性

### 2. 分层架构

**优势**:
- 模块解耦，易于维护
- 支持设备替换(通过HAL接口)
- 便于单元测试
- 适合团队并行开发

### 3. 状态机控制

**优势**:
- 流程可控，防止异常状态
- 便于调试和日志追踪
- 支持断点续扫
- 符合产业化要求

### 4. 事件驱动

**优势**:
- 模块间松耦合
- 支持异步处理
- 便于日志和监控
- 易于扩展功能

## 数据流

### 扫描数据流

```
1. 用户配置扫描参数
   ↓
2. ScanService配置FPGA PSO
   ↓
3. 相机进入硬触发等待
   ↓
4. XY台开始匀速运动
   ↓
5. FPGA监控编码器位置
   ↓
6. 到达触发点→输出TTL
   ↓
7. 相机自动曝光采集
   ↓
8. 图像传输到上位机
   ↓
9. ScanService收集数据
   ↓
10. AlgoService处理数据
   ↓
11. DataService保存结果
```

### 控制流

```
UI层 → 服务层 → 核心层 → HAL层 → 硬件
 ↑       ↑       ↑       ↑       ↑
 └───────┴───────┴───────┴───────┘
        事件总线反馈
```

## 扩展性设计

### 支持瞬态检测

**当前架构兼容性**:
- HAL接口预留时间域采集接口
- FPGA逻辑可重配置
- 状态机支持扩展状态

**升级路径**:
```
当前: Basler相机(ms) → 瞬态: 高速采集卡(GS/s)
当前: LED光源(连续)  → 瞬态: 飞秒激光(脉冲)
当前: DMD(ms)       → 瞬态: DMD+振镜(μs)
```

### 多设备支持

通过HAL接口，轻松支持:
- 不同品牌运动控制卡
- 不同型号相机
- 国产替代设备

### 分布式部署

- 服务层可拆分为独立进程
- 通过ZeroMQ/gRPC通信
- 支持多机协同
